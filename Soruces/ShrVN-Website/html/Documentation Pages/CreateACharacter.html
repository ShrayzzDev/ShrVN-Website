Define a character

To define a character, you have to be in the Character.shrvn file at the root of the project.
You have to create a character variable like that : 

    
      Character instance_name("In-Game_name")

Or like that : 

    
      Character instance_name("In-Game_name",red_value,green_value,blue_value)

instance_name :

  • It's what the program will use to identify the character later in the code.
    Two different characters CAN'T have the same instance name
    For example, in the main script you will show the dialogue of a character like that: 
    
      instance_name : "Things to say"

In-Game_name :

  • It's the name is going to use when showing the dialogue of a character.
    You can't leave it empty or not put it. It is mandatory
    It's also the name that's going to be used to create the associated character folder in the /images/.
    Even if two different character can't have the same instance name, they can have the same In-Game_name.
    But be weary of the fact that they will use the same folder in the /images/ folder.

red_value, blue_value and green_value:

  • These are the color codes used by the engine when showing the name of a character when talking.
    They can't go above 255 (or FF in hexadecimals).
    You also can't only put just one, you need to else precise none (first way of instantiating a character
    Or precise them all.
    NOTE : If you want to put the code in hex, put 0x in front of the number, or else the engine is going
    to take it as a decimal, as it is what is reads by default. You can do one in hex and the others in decimal,
    and it shouldn't make issues. But it is not recommended as it will make your code more confusing.

The only difference between the two ways of declaring a character, is that in the first one you don't precise color codes,
and in the second one you do. By default, (so in the first case), the color codes are all 0, the chracter name will be black.
