First, let's talk about how the files are structured.

In a ShrVN project, you'll find two directories :

  • The images directory
  • The init directory

The image directory is pretty self-explanatory, it will contain the images that will be used by the engine.
Inside it, the project will also make a directory for each character, but we will talk about that deeper later
The init directory will contain all the files needed to create the layout of the menus, screens, etc . . .
Don't worry, these have base values and you wil not have to set them yourself.
If you happen to delete a line by accident, that won't break anything, technically you could delete the init folder
and the engine would still work, but I wouldn't recommend to do that, even if you don't change the values.
(this will be detailed in the full documentation, here we'll focus on how to get a basic novel to run)

In the root folder, there's another very import file, the "Characters.shrvn". It will be used to define Characters.
So, let's talk about how ShrVN handles characters

Characters

Here's an example of a character definition : 

    
      Character jean("Jean")

This is the easiest way to create a Character.
Here, only the name showed in-game is precise, the base color is black #000000
The in-game name is Jean, and the instance name is jean. The difference is important.
The instance name is the one you're going to use when the character is talking in the main script, like that: 

    
      jean : "My name is Jean"

You can also use the same name and instance name if you want, that won't cause any problems.
Also, it is very recommended to use at least 2 letters instance name, for it to be clearer
If you want to precise the color of the character, you can define the character like that: 

    
      Character jean("Jean",25,25,25)

You obviously need to precise the code of each color, if not the game will crash.

Now, let's talk about images.

Images in Characters

Each character has its own folder in the images folder.
For example, here for Jean, when adding images it will look in the Jean folder, in the images folder.
When instantiating a character, it will automatically create the folder, but you can create it yourself if you want.
So, you may create all character, launch the game a first time for it to create all folders, and then put the images in it.
NOTE : The name to the folder is the in-game character name NOT THE INSTANCE NAME.
Each character has a list of images. You can add an image to a character like that: 

    
      From jean Add smiling.png as Smiling

After the 'From', you put the instance name of the character.
After the Add you put the path to the image, from the images/character_name folder.
After the As, you put the name that will be used when showing the image in the main script.
